搭建你的个人博客！

发表于 2022-07-26 阅读次数：

这是一篇教你如何搭建个人博客的文章

一. 首先需要利用GitHub

![](/Users/anne_lister/Desktop/screenshot/截屏2022-07-26 22.05.21.png)

创建一个为yourusername.github.io的仓库

这个仓库名一定要是自己的用户名字，设置为public，不要勾选那个README !!!

二. 下载node、git

这边建议去node官网下载，要包括npm。

![截屏2022-07-26 22.09.53](/Users/anne_lister/Desktop/screenshot/截屏2022-07-26 22.09.53.png)

git可以在官网下载，用Mac的可以用brew下载

下载完node、npm、git以后，检查一下版本

![截屏2022-07-26 22.15.26](/Users/anne_lister/Desktop/screenshot/截屏2022-07-26 22.15.26.png)

三. 下载hero

npm install hexo-cli -g

用这个指令下载hexo框架，完事以后出现报错 —>>>

![截屏2022-07-26 22.18.42](/Users/anne_lister/Desktop/screenshot/截屏2022-07-26 22.18.42.png)

搜了一下，是因为没有sudo权限，所以我们需要用这个指令

sudo chown -R $(whoami) $(npm config get prefix)/{lib/node_modules,bin,share}

执行指令需要输入本机密码，然后再重新执行npm install hexo-cli -g这个指令，hexo就下载好了

四. 初始化以及配置

然后我们需要在一个空的文件夹初始化一下，就像GitHub初始化仓库一样，这个文件夹就是你网站的仓库

执行指令hexo init XXXXXXXX.github.io 注意这里XXXXX是你的用户名

文件夹中多出了这个github.io的文件夹，点进去，就是你hexo的一个小仓库了。

进去你的文件夹，试着执行hexo server

![截屏2022-07-26 22.25.21](/Users/anne_lister/Desktop/screenshot/截屏2022-07-26 22.25.21.png)

下面的网址 http://localhost:4000/就是你的网站，但是此时只是在本地，别人不可以访问。

五. 在GitHub上部署网址，别人就可以访问了

在部署之前，得先在本地生成ssh密钥，为什么要这么做？如果不这么做就会出现这样的报错err: Error: Spawn failed

进入你的/.ssh目录，cd ~/.ssh

ssh-keygen -t rsa -C "your_email@youremail.com"

用这行命令，然后连续输入回车，最后会生成这样的乱码：

![截屏2022-07-26 22.29.33](/Users/anne_lister/Desktop/screenshot/截屏2022-07-26 22.29.33.png)

然后执行这个指令cat id_rsa.pub

会生成你的密钥，全部复制下来，在GitHub设置上添加密钥

![截屏2022-07-26 22.33.35](/Users/anne_lister/Desktop/screenshot/截屏2022-07-26 22.33.35.png)

配置hexo文件夹的config.yml文件：

![截屏2022-07-26 22.36.07](/Users/anne_lister/Desktop/screenshot/截屏2022-07-26 22.36.07.png)

这里repo的内容是你在第一步新建仓库的ssh，好像http链接也可以，可以试试。

接下来执行hexo d指令，意思就是display到GitHub。

如果你和我一样出现这段报错：

![截屏2022-07-26 22.39.50](/Users/anne_lister/Desktop/screenshot/截屏2022-07-26 22.39.50.png)

请执行这些指令：

rm -rf .deploy_git

git config --global core.autocrlf false

hexo clean

hexo g

hexo d

如果一路绿灯就说明部署好了！这个hexo d指令容易报错，多执行几次试试

如果你的网站打不开的话，在GitHub仓库设置一下pages

![截屏2022-07-26 22.46.58](/Users/anne_lister/Desktop/screenshot/截屏2022-07-26 22.46.58.png)

我这个是已经设置好的，刚点进去一个是有个save，点击后等一会，然后就像我这张图一样出现绿勾勾，那就是部署好了。

如果你的网站还是404，检查一下仓库名到底是不是用户名，大小写不一样都不行。

最后，网站部署完后的样子是这样的：
