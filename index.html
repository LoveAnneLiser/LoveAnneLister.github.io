OUC_CSer自学指南

发表于 2023-09-04

前言

: ) 一年前曾经在pyq说过：“等我变强了要写一个OUC CSer自学指南”，然而一年过去了实际上我也没有变强多少，只是会一些CRUD罢了。写这个的主要目的是想帮助后面的同学更好地认识CS，不至于说讨厌（因为我知道不少同学来了这个专业已经打算转行了）。再有一个目的是，帮助大家扎实基础，侯捷老师说过“勿在浮沙筑高台”，祝大家在CS的道路上越走越好。

学习路线

注：本路线不涉及公共基础课

大一

广度认知

刚入学学的是《计算机科学导论》，其实现在想想，确实是一门不错的课，但一些课程作业真的有点太难了。我这里推荐学习《Crash Course Computer Science》，B站有视频。https://www.bilibili.com/video/BV1EW411u7th?vd_source=99690035e06ef0ba562b557f372ceec0。主要是从各个角度理解计算机，教给学生一个CS MAP，虽然讲的不深，但是一定程度上扩产了一个小白的计算机知识。

《MIT-Missing-Semester》：教你一些计算机常用的工具，git、shell等 https://www.bilibili.com/video/BV1x7411H7wa?vd_source=99690035e06ef0ba562b557f372ceec0

编程语言

学习方法：学完一门编程语言，最重要的是要用起来，计算机的出现是为了解决需求，所以学完了可以去GitHub找一个小项目进阶。

C语言

让一个刚入门CS的学生就上手C语言，实在感觉有点不合适。在四大(MIT, CMU, UCB, Stanford)的入门课程中，一般都是使用python入门。

我在这里推荐下UCB的CS61A https://cs61a.org/

也可以看浙大的C语言课程 https://www.bilibili.com/video/BV1dr4y1n7vA?vd_source=99690035e06ef0ba562b557f372ceec0

C++

我推荐的C++课程是侯捷老师的C++！Youtube有他的课！他的主要路线是C++面向对象—> C++ STL —> C++新标准 C++11 —> C++内存管理 —> C++设计模式
有条件的同学可以去看一下，真的讲的不错。
C++书籍：《C++ primer》 配合B站的课程看。

离散数学

我个人觉得这门课很硬，对未来的课程理解很有帮助。
建议看北大的课程 https://www.bilibili.com/video/BV1BW411n7gw?vd_source=99690035e06ef0ba562b557f372ceec0
一位太原理工大学的老师讲的也非常不错 https://www.bilibili.com/video/BV1Z5411h7gb?vd_source=99690035e06ef0ba562b557f372ceec0

操作系统实验

实验一

实验内容

  1. 编写一C语言程序（程序名为fork.c），使用系统调用fork( )创建两个子进程。当程序运行时，系统中有一个父进程和两个子进程在并发执行。父亲进程执行时屏幕显示“I am father”，儿子进程执行时屏幕显示“I am son”，女儿进程执行时屏幕显示“I am daughter”。
  2. 多次连续反复运行这个程序，观察屏幕显示结果的顺序，直至出现不一样的情况为止。记下这种情况，试简单分析其原因。

实验目的

  1. 了解进程与程序的区别，加深对进程概念的理解；
  2. 进一步认识进程并发执行的原理，理解进程并发执行的特点，区别进程并发执行与顺序执行；
  3. 分析进程争用临界资源的现象，学习解决进程互斥的方法。
  4. 了解fork( )系统调用的返回值，掌握用fork()创建进程的方法；
  5. 熟悉wait、exit等系统调用。

习题

(1)套接字在网络编程中的作用是什么？为什么称它为套接字？

作用：简单来说，就是表示两台计算机之间的网络连接。

称为套接字的原因是，为了与远程计算机进行数据传输，需要连接到网络，编程中的套接字就是用来连接网络的。

(2)在服务器端创建套接字后，会依次调用listen函数和accept函数，请比较并说明二者作用。

listen函数是为了能够让套接字转化为可连接的状态，意味着套接字可以收到连接请求

accept函数是在收到连接请求之后，选择要不要接受请求。

(3) Linux中，对套接字数据进行I/O时可以直接使用文件I/O相关函数；而在Windows中不可以，原因为何？

在Linux中，有着一切皆为文件的想法，就像socket都可以用read/write读写一样。

Windows将文件与套接字I/O分开，是为了保持自然差异性。

(4) 创建套接字后一般会给它分配地址，为什么？为了完成分配需要调用什么函数？

只有分配了地址以后，其它的端口/客户端/服务端才能找到自己，分配的地址为IP地址和端口号，网络就是通过IP地址和端口号确认路径的。为了完成分配需要调用bind函数

(5) Linux中的文件描述符与Windows的句柄实际上非常相似。请以套接字为对象说明它们的含义。

(6) 底层文件I/O函数与ANSI标准定义的文件I/O函数之间有何区别？

(7) 参考本书给出的实例low_open.c和low_read.c，分别利用底层文件I/O和ANSI标准I/O编写文件复制程序。可任意指定复制程序的使用方法。

虚拟地址空间

linux系统会给当前进程分配一个2^32大小的一块空间，也就是4G的大小

先从用户空间讲起，用户空间分配了3G的大小

  • .text：用来存放指令，已编译的机器代码
  • .rodata：只读数据，就像 const int N = 110
  • .bss：用来存放未曾定义过或者是初始化为0的数据
  • .heap：malloc生成的数据就在堆上
  • 加载共享库
  • .stack
  • 命令行参数和环境变量

内核空间，分配了1G的大小，ZONE_DMA ZONE_NORMAL ZONE_HIGHMEN在内核空间

每个进程的用户空间是私有的，但是内核空间是共享的。(比如进程间通信)
